AllPick

역할 기반 JWT 인증 흐름에서 Role 정보 불일치가 발생한 문제

JWT · Redis · Spring Security expand_more

원인

member의 role 구조가 변경됐지만 로그인 코드에는 기존 role 기준 로직이 남아 있어, JWT 발급 이후 권한 판단이 실제 계정 구조와 어긋났습니다.

해결

사용하지 않는 role 의존 로직을 제거하고 구매자·판매자·관리자 인증 흐름과 토큰 저장 방식을 분리했습니다.

TFT-gogo

관리자 수동 집계 요청 후 실제 작업 상태와 화면 상태가 어긋난 문제

Async · Executor · State expand_more

원인

HTTP 요청 성공과 실제 비동기 작업 등록·실행 상태를 구분하지 않아, 중복 요청이나 executor 등록 실패도 성공처럼 보일 수 있었습니다.

해결

중복 실행과 작업 등록 실패를 별도 응답으로 분리하고, 실패 시 실행 중 플래그가 복구되도록 상태 계약을 정리했습니다.

TFT-gogo

매치 집계 중 Riot API 요청이 키 한도를 초과할 수 있던 문제

Riot API · Rate Limiter · Queue expand_more

원인

Riot API 호출 경로가 RiotApiClient와 RiotQueue로 나뉘어 있었고, 매치 수집 경로만 더 짧은 간격으로 요청해 API 한도를 초과했습니다.

해결

모든 매치 수집 요청을 단일 rate limiter로 통합하고, 큐 크기 제한과 종료 처리 기준을 추가했습니다.
